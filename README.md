# Project Janus

> **VR Neuropsychological Assessment Simulation**  
> Developed in collaboration with the UNT Psychology Department's Neuropsychological Innovation Lab (NIL)

---

## Overview

Project Janus is an immersive VR simulation designed to improve the assessment of memory and real-world functioning in individuals with dementia and other cognitive impairments. The simulation presents patients with realistic, everyday virtual environments to elicit and evaluate cognitive responses in a standardized, repeatable way.

## Quick Start

### 1. Clone the Repository

1. Download and install [GitHub Desktop](https://desktop.github.com/download/)
2. Sign in with your GitHub account
3. Clone the Project Janus repository to your local Unity projects folder

### 2. Install Unity

1. Download [Unity Hub](https://unity.com/download) and sign in
2. In Unity Hub → **Installs** → **Install Editor** → select version **3.3.1**
3. Install Visual Studio Code when prompted during the editor setup

### 3. Open the Project

1. In Unity Hub → **Projects** → **Add project**
2. Select the cloned repository folder
3. If prompted about a missing editor version, install the recommended version before opening

### 4. Install Required Packages

When Unity opens the project for the first time:
1. Accept all package install prompts
2. In **Project Settings → XR Plug-in Management**, install any missing XR plugins
3. Install the **Meta XR Simulator** from your Downloads folder when prompted
4. Go to **Meta → Meta XR Tools** and resolve any remaining issues

### 5. Run the Simulation

1. In the **Project** panel, navigate to `Assets → Scenes`
2. Double-click **MainMenu** to load the entry scene
3. Click the **Game** tab → press **Play (▶)**

### Running Without a VR Headset

1. Go to **Meta → Meta XR Simulator → Activate**
2. Press **Play** — the Meta XR Simulator app will launch
3. Use **WASD** to move and **mouse** to interact
4. Enable **Pointer Mode** to simulate controller inputs

## Project Structure

```
ProjectJanus/
├── Assets/
│   ├── Scenes/          # Unity scene files (.unity) — entry point: MainMenu.unity
│   ├── Scripts/         # C# game logic and assessment scripts
│   └── Prefabs/         # Reusable Unity GameObjects
├── Packages/            # Unity package manifest (auto-managed)
├── ProjectSettings/     # XR plugin config, input mappings, quality settings
├── UserSettings/        # Local editor preferences (not committed)
└── .gitignore
```

> **Note:** The `Library/` and `Temp/` folders are auto-generated by Unity locally and are **not** stored in the repository.

## Troubleshooting

**Android Player installation error**  
Navigate to `C:\Program Files\Unity\Hub\Editor\<version>\Editor\Data\PlaybackEngines\` and delete the `AndroidPlayer` folder. Uninstall the affected editor version in Unity Hub, restart your computer, then reinstall.

**Missing packages on first open**  
Unity will prompt you to install missing packages automatically. Click **Install** on each prompt and wait for completion before running the project.

**Wrong editor version**  
Always use Unity Editor **3.3.1**. Using a newer or older version may break scenes or scripts.

## Notes

- This project is built and tested for the **Meta Quest 3**. Other headsets (HTC Vive, Valve Index, etc.) require additional XR plugin configuration.
- No external database or manual data initialization is required.
- Runtime session data is stored locally at: `C:\Users\<username>\AppData\LocalLow\<Company>\ProjectJanus`
- No credentials, API keys, or secrets are stored in the repository.
